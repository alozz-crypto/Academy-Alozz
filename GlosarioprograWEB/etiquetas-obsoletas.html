Etiquetas Obsoletas en HTML5

Estas etiquetas no deben usarse en nuevos proyectos. Han sido reemplazadas por alternativas modernas.

Etiquetas de Formato Obsoletas

<font face="Arial" size="4" color="red">Texto</font>
<center>Texto centrado</center>
<strike>Texto tachado</strike>
<u>Texto subrayado</u>

Alternativas con CSS:

<p style="font-family: Arial; font-size: 1rem; color: red;">Texto</p>
<p style="text-align: center;">Texto centrado</p>
<del>Texto tachado</del> o <span style="text-decoration: line-through;">...</span>
<span style="text-decoration: underline;">Texto subrayado</span>

Otras Etiquetas Desaprobadas

<frameset>
  <frame src="menu.html">
  <frame src="contenido.html">
</frameset>
<applet code="MiApplet.class"></applet>
<marquee>Texto que se mueve</marquee>

Alternativas modernas:

Usar <iframe> o diseño CSS moderno en lugar de frames
Usar <object> o tecnologías modernas (JavaScript) en lugar de applets
Usar animaciones CSS o JavaScript para efectos de movimiento

¿Por qué se volvieron obsoletas?

  • Separación de preocupaciones: HTML para estructura, CSS para presentación
  • Problemas de accesibilidad: Muchas etiquetas antiguas causaban problemas con lectores de pantalla
  • Mantenimiento: Las alternativas modernas son más fáciles de mantener
  • Rendimiento: Algunas etiquetas como <blink> consumían muchos recursos
  • Estandarización: HTML5 buscó simplificar y unificar el lenguaje
